# Project Statement – Personal Expense Tracker

## 1. Problem Statement

Managing money becomes challenging when expenses are scattered across different places like food, travel, bills, and small daily purchases. Many people rely on memory or rough notes, which often leads to confusion at the end of the month. Without a structured system, it is difficult to understand spending patterns, stick to a budget, or make informed financial decisions. This project aims to provide a simple tool that keeps all expenses organized in one place and helps users stay within their planned budget.

## 2. Scope of the Project

The project focuses on developing a command-line based application that handles essential expense-tracking tasks. It allows users to record expenses, assign categories, and manage monthly budget limits. The system also supports storing records in a local database and generating text-based reports for reference.  
The scope is intentionally kept limited to personal use and small-scale expense management. It does not include complex financial forecasting, online payments, multi-user login, or advanced analytics.

## 3. Target Users

- College students who need a simple way to track their spending  
- Individuals who prefer offline, lightweight tools instead of mobile apps  
- Beginners practicing Java concepts like OOP, file handling, exceptions, multithreading, and JDBC  
- Anyone who wants a basic record of monthly expenses without relying on external apps  

## 4. High-Level Features

- Add new expenses with description, amount, category, and payment method  
- Set monthly and category-based budgets  
- View expenses in different formats: all, category-wise, and summary view  
- Simple warnings when the user crosses a predefined budget  
- Store expenses permanently using SQLite  
- Export data to CSV and generate text reports  
- Maintain a short history of user operations  
- Background report generation using multithreading
